## Category Master Report: Equational Theory (FP Focus)

This report synthesizes multiple batch analyses identifying why models incorrectly prove implications (False Positives) or fail to identify invalid logic in the context of abstract magmas.

### **Core Pillars of Logical Strategy**
1.  **Strict Syntactic Substitution:** Valid derivations must treat equations as rigid trees. Every step must be a 1-to-1 replacement where a sub-term exactly matches an axiom's structure.
2.  **Exhaustive Finite Model Verification:** A counterexample is only valid if *every* cell in the Cayley table is checked against *every* variable combination $(x, y, z \dots)$ in the equations.
3.  **Variable-Value Binding:** In complex evaluations (Depth $\geq$ 3), success requires explicit "binding blocks" (e.g., `Let x=0, y=2, z=1`) to prevent variable drifting.
4.  **Invariant Identification:** Identifying properties that the source equation *cannot* change (e.g., word length, variable parity, or depth) to provide a "Negative Proof" of non-implication.

---

### **Critical Failure Points (Hallucination & Pitfalls)**
1.  **The "Global Identity" Fallacy:**
    *   *Error:* Assuming that because a term $T$ satisfies $x * T = x$, $T$ is a universal identity $e$.
    *   *Pitfall:* Treating $T$ as a movable constant or assuming $T * x = x$ without a separate derivation.
2.  **Illegal Structural Generalization (Stealth Properties):**
    *   *Error:* Subconsciously applying Commutativity ($a*b = b*a$) or Associativity ($(a*b)*c = a*(b*c)$) to "bridge" structurally different equations.
    *   *Pitfall:* Treating the output of a complex sub-expression (the image) as a free, universally quantified variable.
3.  **Symbolic Overload (The Depth-3 Collapse):**
    *   *Error:* At nesting depths $\geq 3$ (e.g., `x * (y * (z * w))`), the model loses track of variable positions.
    *   *Pitfall:* "Variable Dropping," where the model simplifies a 4-variable term into a 2-variable term by "vanishing" middle variables that don't appear in the target equation.
4.  **Arithmetic & Modular Bias:**
    *   *Error:* Assigning integer properties (like $x+1$ or $x-1 \pmod n$) to magma operations.
    *   *Pitfall:* Hallucinating a "Phantom Rule" (e.g., assuming $x * x = x$ or $(x+1)*x = 1$) to force a calculation to match a desired verdict.
5.  **The Verdict-Reasoning Mismatch:**
    *   *Error:* The model identifies a contradiction in its math (`0 != 2`) but panics and outputs `VERDICT: TRUE` or a `PROOF:` block regardless.

---

### **Actionable Logic & Refutation Heuristics**

**Derivation Rules (For TP):**
*   **Rule of Exact Match:** If Eq1 is $x = x * (y * z)$, you may only replace a term if it follows the $(Variable) * (Term1 * Term2)$ pattern.
*   **Depth-Jump Requirement:** If Eq1 has $x$ at depth 1 and Eq2 has $x$ at depth 4, the proof *must* explicitly show three distinct substitution steps that increase nesting depth.

**Refutation Triggers (For TN/FP Prevention):**
*   **The "New Variable" Trigger:** If Eq2 introduces a variable (e.g., $w$) not present in Eq1, and Eq1 does not provide a way to create a constant or identity, the implication is likely **FALSE**.
*   **The "Independence" Check:** If the RHS of Eq2 contains a variable $z$ that the LHS does not, Eq1 must prove that the entire RHS expression is invariant under $z$. If no such "absorption" rule exists in Eq1, refute the proof.
*   **Table Arithmetic Check:** If a counterexample is proposed, immediately calculate the innermost parenthesis of the first failing case. If `table[a][b]` does not equal the model's claim, discard the entire derivation.

**Strict Constraints for System Prompts:**
*   **Constraint:** DO NOT use "Existence holds" or "No logical barrier" as a substitute for a Cayley table.
*   **Constraint:** DO NOT skip steps. Every line must cite: `Sub [var] = [term] in Eq1`.
*   **Constraint:** Evaluate from the **innermost** parentheses outward. Never skip to the result.

### **Key Logical Patterns (Successful strategies / Foundational Rules)**
*   **Intermediate Variable Extraction:** To bypass the model's complexity threshold, successful derivations strictly map Abstract Syntax Trees (AST) from the inside out. For any equation nested > 2 levels, intermediate variables must be explicitly defined (e.g., Let $T_1 = y * z$, Let $T_2 = T_1 * w$).
*   **Strict Independent Evaluation:** Valid proofs evaluate the LHS and RHS completely independently using forward-computation. They never assume LHS = RHS prematurely or substitute fragments of the target equation into the premise.
*   **Axiomatic Blank Slate:** Treat the generic magma operation `*` strictly. Do not apply associativity (`(x*y)*z = x*(y*z)`), commutativity (`x*y = y*x`), or idempotence (`x*x = x`) unless explicitly dictated by the provided equation. 
*   **Exhaustive Universal Quantifier Strictness:** When verifying "for all $x,y,z$" on finite models, absolute verification requires evaluating every single cell combination, explicitly alternating standard elements and special elements (e.g., $x=\omega, y=1, z=\omega$). 

### **Critical Failure Points (Specific pitfalls/hallucinations to avoid)**
1.  **Symbolic Overload (The "Depth-3" Breakdown):** The most frequent failure. When equations exceed a nesting depth of 3 (e.g., `(((y*x)*y)*z)*x`), the model's working memory collapses. It drops variables, invents algebraic properties, or hallucinates universal claims (e.g., "this always evaluates to $x$") instead of computing the math.
2.  **Cayley Table Blindness & "Amnesia":** The model will construct a valid rule or finite table (e.g., `1 * 1 = 0`), but instantly hallucinate contradictory lookups (e.g., claiming `1 * z = 1` for all $z$) to force a desired equation to hold. 
3.  **Phantom Reductions & Variable Truncation:** Illegally dropping terms during substitution to force a match. *(Example: If Rule 1 is $A * 0 = A$, the model evaluates $(x * y) * 0$ and hallucinates the output as $x$ instead of the mathematically strictly required $(x * y)$.)*
4.  **"Magic Wand" & Circular Constructions:** Attempting to prove a counterexample by asserting it exists without doing the math. The model uses hand-wavy logic like "configure the remaining elements so Eq 1 holds" or circularly builds a "quotient magma by construction" without verifying if it collapses.
5.  **Ill-Defined Piecewise Magmas (Domain Collisions):** Creating custom infinite magmas with multiple conditional rules that overlap (e.g., Rule 1 applies to "all integers", Rule 2 applies to "1"). This causes operations to yield multiple conflicting outputs for the same input.
6.  **Variable-Name Dependence:** A fundamental algebra hallucination where the model defines operations based on the *variable's letter* (e.g., $w * x = 2x$) rather than treating them as universally quantified placeholders.

### **Refutation Heuristics (When to stop proving and start disproving)**
*   **The "Hand-Waving" Trigger:** Immediately abort and label as failed logic if the text uses phrases like "the detailed construction is complex, but existence follows," "Mace4 can complete this," or "a concrete counterexample exists." 
*   **The Goal-Directed Retrofit Trigger:** Halt verification if the proof assumes the conclusion (LHS = RHS) and reverse-engineers the intermediate table lookups or inverse operations (e.g., $t*t = \lfloor t/2 \rfloor$) to bridge the gap. 
*   **The Verdict/Format Disconnect:** If the model text successfully generates a `VERDICT: FALSE` (refuting the implication via counterexample) but outputs a final boolean or system flag of `TRUE`, override the flag. The model has hallucinated its formatting, not its text.
*   **Fractional/Non-Closed Operations Trigger:** If a custom magma over integers or natural numbers yields fractional, unhandled, or out-of-bounds outputs (breaking mathematical closure), the counterexample is immediately invalid.

This synthesis identifies the recurring logical patterns and critical failure points observed across multiple batch reports for the **FP_1 (False Positive) Category** in equational theory challenges.

### **Core Pillars of Successful Derivation (TP/FN)**
1.  **Atomic Substitution:** Breaking down equations with nesting depths $>3$ into intermediate variables (e.g., $v_1 = w * x, v_2 = z * v_1$) to prevent "context dropping" and variable leakage.
2.  **Symmetry Auditing:** Explicitly verifying if an operation is commutative ($x*y = y*x$) or associative ($x*(y*z) = (x*y)*z$) before shifting any term or parenthesis. If not explicitly given in `eq1`, these properties are treated as non-existent.
3.  **Universal Quantifier Mapping:** Treating every variable in `eq1` as a placeholder that must be replaced *simultaneously* and *completely* across the entire equation during substitution.

---

### **Critical Failure Points (Avoidance List for FP/FN)**
1.  **The "Right-Identity" Hallucination:** Assuming $x * f(y) = x$ implies $f(y)$ is a universal identity $e$. In magmas, $f(y)$ might only result in $x$ for a specific structure; it cannot be substituted for $x$ in other contexts without a global proof.
2.  **"Table-Fudging" & Cayley Cheats:** Constructing a finite magma table but "lying" about specific cell results (e.g., defining $1*1=0$ in the table but claiming $1*1=1$ during a proof step) to force an equation to balance.
3.  **Circular Verification (Begging the Question):** Assuming `eq2` is true and performing operations on both sides to reach $x=x$. This proves `eq2` is self-consistent, not that it is derived from `eq1`.
4.  **Complexity Threshold Collapse:** Beyond 3 levels of nesting or 4 unique variables, models revert to "Hand-Waving" (e.g., "The structural heaviness suggests independence") instead of algebraic derivation.
5.  **Target Variable Fixing:** Attempting to prove `eq2` by setting its free variables to specific constants (e.g., "Let $u = w$"). A valid proof must hold for all arbitrary values of variables in `eq2`.

---

### **Refutation Heuristics (When to stop proving and start disproving)**
*   **Structural Mismatch Trigger:** If `eq1` is strictly left-nested (e.g., `((x*y)*z)`) and `eq2` is strictly right-nested (e.g., `(x*(y*z))`), and no associativity is provided, the implication is likely **FALSE**.
*   **The Projection Test:** Test if the **Left-Projection Magma** ($x * y = x$) or **Right-Projection Magma** ($x * y = y$) satisfies `eq1` but violates `eq2`. If either projection creates a contradiction, the implication is **FALSE**.
*   **Variable Independence Check:** If `eq2` introduces a variable (e.g., $w$) that appears in a position where it cannot be "absorbed" or "cancelled" by the specific structures in `eq1`, the implication is likely **FALSE**.
*   **The Sink-Variable Trap:** If a derivation leads to $x = y$ (collapsing the magma to a single element), the logic has likely hallucinated a "sink" property. Re-evaluate; if $x=y$ is not a necessary consequence of `eq1`, the implication is **FALSE**.

---

### **Actionable Logic (Derivation Rules)**
*   **Rule of Simultaneous Replacement:** If substituting $x \to (a*b)$, every instance of $x$ in the axiom must change to $(a*b)$ in the same step. No partial substitutions.
*   **Rule of Parenthetical Strictness:** $(A*B)*C \neq A*(B*C)$. Any step that moves a parenthesis without a cited associativity lemma is an **ILLEGAL STEP**.
*   **Rule of Concrete Verification:** A counterexample is only valid if a **complete** Cayley table is provided and **every** permutation of elements is tested against `eq1`. "Pattern-based" or "Hand-wavy" magmas are discarded.

Here is the synthesized Category Master Report, distilling the individual batch analyses into a high-density, actionable framework for evaluating equational theories.

### Key Logical Patterns (Successful strategies): (For TP/FN)
*   **Strict Universal Instantiation:** Treat every variable as a universal placeholder. A single, isolated variable (e.g., $w$) can and must be able to represent any deeply nested product structure. 
*   **Rigorous Step-by-Step Substitution:** Valid proofs rely **only** on three operations: direct substitution of the given axiom (Eq1), replacement of equal subterms, and transitivity. Prose descriptions of "structural properties" (e.g., "nested prefixes force an identity") are not proofs.
*   **Intermediate Variable Anchoring:** To survive the "Complexity Threshold" (equations with >4 variables or >3 nesting levels), chunk complex sub-expressions using intermediate variables (e.g., "Let $k = y * z$"). This preserves structural integrity, prevents variable erasure, and keeps anchor variables strictly separated from parameter variables.
*   **Contextual vs. Global Properties:** Correctly identify when an element acts as an identity or constant *only within a specific nested context* versus being a global property applicable to the entire domain.

### Critical Failure Points (Specific pitfalls/hallucinations to avoid): (For FP/FN)
*   **Phantom Axioms (Illegal Algebra):** The most common hallucination is implicitly assuming commutativity ($x*y = y*x$) or associativity ($x*(y*z) = (x*y)*z$) in a raw magma. Never reorder variables or shift parentheses unless Eq1 explicitly permits it.
*   **The Identity / Constant Collapse:** Erroneously assuming that repeated variables (e.g., $z*z$) evaluate to a universal constant ($c$, $0$, $1$) or that a complex sub-term "absorbs" other variables to become an identity element without an explicit derivation.
*   **Circular Reasoning (Target-to-Source Backtracking):** Using the target conclusion (Eq2) as a justified substitution step during the derivation, or assuming Eq2 is true to force a simplification on the right-hand side.
*   **Symbolic Overload & Black-Boxing:** When nesting depth exceeds 3, models often collapse complex terms into generic "black boxes" (e.g., treating $((y*z)*w)$ as a static dummy variable that cancels out), ignoring the fact that universal quantifiers apply recursively to inner terms.
*   **Verdict/Verification Mismatch:** Reaching a logical conclusion of "FALSE" in the Chain-of-Thought (e.g., via a counterexample attempt) but outputting a final metadata verdict of "TRUE". 

### Refutation Heuristics (When to stop proving and start disproving): (For TN)
*   **Mandatory Exhaustive Cayley Tables:** If attempting a finite counterexample, the Cayley matrix must be 100% complete ($N \times N$). Every single variable permutation ($\forall x,y,z...$) must be manually verified against Eq1. Reject "partial tables" or tables relying on isolated "glitch points."
*   **Reject "Unreachable Element" Defenses:** Equational logic is universally quantified. Stop using arguments that partition the domain (e.g., "if $x$ is even" or "element $d$ is never produced by a product"). If Eq1 holds, it holds for all elements, regardless of their origin.
*   **Syntactic Mismatch $\neq$ Disproof (Free Magma Fallacy):** Never conclude that Eq1 does not imply Eq2 simply because they have different variable counts, reuse variables differently, or possess different left/right nesting structures. Visual mismatch is not a disproof; refutation strictly requires a mathematically consistent counter-model.
